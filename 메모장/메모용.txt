ros2 launch humanoid_bringup ak70_driver.launch.py
이 launch 파일이 다음 노드를 실행한다.
Node(
    package='humanoid_control',
    executable='ak70_can_node',
)

즉 실행되는 프로그램은
ak70_can_node.py


ㅡㅡㅡㅡㅡㅡㅡㅡㅡROS2 노드 초기화

rclpy.init()
node = AK70CanNode()
rclpy.spin(node)

파라미터 선언
declare_parameter()

ㅡㅡㅡㅡㅡㅡㅡㅡㅡ사용되는 주요 파라미터

parameter
can_interface	CAN 인터페이스
motor_id	모터 ID

kp	위치 제어 gain
kd	위치 제어 damping

ㅡㅡㅡㅡㅡㅡㅡㅡㅡ모터 초기화
노드 시작 직후 실행됨

모터 모드 진입
enter_motor_mode()

보내는 CAN frame

FF FF FF FF FF FF FF FC
초기 zero command
zero_command()

즉

p = 0
v = 0
kp = 0
kd = 0
torque = 0

를 여러 번 보내 모터를 초기화

ㅡㅡㅡㅡㅡㅡㅡㅡㅡ제어 루프 생성
self.control_timer = self.create_timer(
    control_period_sec,
    self.control_timer_callback
)

현재 설정

control_period_sec = 0.02

= 50 Hz control loop

명령 토픽 수신
외부에서 다음 명령을 보내면

ros2 topic pub /control_command std_msgs/msg/Float64 "{data: 1.0}"

콜백이 실행

control_command_callback()

이 함수는 단순

self.target_position = msg.data
self.last_command_time = now

target_position 업데이트만 수행

ㅡㅡㅡㅡㅡㅡㅡㅡㅡ제어 루프 실행 (50Hz)
50Hz마다 실행되는 함수

control_timer_callback()

여기서 실제 모터 제어가 수행

timeout 체크
elapsed_sec = now - last_command_time

명령이 오래 안 오면

holding last target

로그만 출력하고 목표값 유지한다.

ㅡㅡㅡㅡㅡㅡㅡㅡㅡR모터 피드백 수신

모터는 매 명령마다 feedback frame을 보냄

읽는 코드

msg = self.bus.recv()

MIT 피드백 디코딩
parse_feedback()

frame 구조

byte0 motor id
byte1~2 position
byte3~4 velocity
byte4~5 torque

이를 다시 float로 변환
